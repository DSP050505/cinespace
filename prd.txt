PROBLEM & GOAL
The core problem
Two people want to watch a movie together remotely — with zero sync delay, both
cameras on, and voice chat. Existing solutions (Teleparty, Kast) have lag, require
browser extensions, or don't support direct file/stream URLs freely.
Success criteria
 Play/pause/seek events sync within <300ms between both users
 
 No audio drift between users over a 2-hour session
 
 WebRTC video+audio call quality ≥ 720p at 30fps
 
 Works on Chrome/Firefox/Safari without any extensions
 
 Room creation to watching in under 60 seconds

SYNC ARCHITECTURE (CRITICAL — READ CAREFULLY)

How zero-lag sync works
The host is the source of truth. All sync events flow through a Socket.io room. The
key insight: don't relay video data — just relay control events (play, pause, seek).
Each user streams the same video URL from their own device, so bandwidth is
never shared.
Flow:
 Host presses play
 Emit SYNC_PLAY + timestamp
 Server broadcasts to room
 Guest receives event
 Adjust for RTT
 Guest plays at exact time
Clock sync algorithm
 On connect: measure RTT between client and server (3 pings, take median)
 
 On SYNC_PLAY: guest seeks to event.videoTime + (RTT/2) then plays
 
 Drift correction: every 10s, compare both users' currentTime via socket; if diff
 >0.5s, silently nudge guest playback with small rate adjustments (1.05x or 0.95x)
— never hard seek
 On seek: emit SYNC_SEEK with target time; guest seeks to exact time + RTT/2
 
 On pause: emit SYNC_PAUSE — both pause instantly, no offset needed
 
 EMIT
sync:play
{"roomId", "videoTime", "serverTime"} — host only
 EMIT
sync:pause
{"roomId", "videoTime"} — host only
 EMIT
sync:seek
{"roomId", "videoTime"} — host only
 EMIT
sync:heartbeat
{"roomId", "currentTime"} — every 10s, both users
 ON
sync:drift
{"delta"} — server tells guest to adjust playback rate
 ON
peer:joined / peer:left
triggers WebRTC offer/answer flow
WEBRTC — VIDEO & VOICE

Implementation with PeerJS
 Use PeerJS — it wraps WebRTC and handles offer/answer/ICE automatically
 
 On room join: get camera+mic via getUserMedia({video: true, audio: true})
 
 Host creates a Peer, shares their PeerID via Socket.io to guest
 
 Guest calls peer.call(hostPeerId, localStream)
 
 Both users show their own video (muted, to avoid echo) and the other's video with
 audio
 STUN: use metered.ca free TURN/STUN credentials (handles NAT traversal —
 works on all networks including hotspots and corporate WiFi)
 Video tiles: small 200x150px floating PiP in corner of screen during movie
 playback
 Mute/unmute and video on/off controls always visible during watch
 
MOVIE SOURCE & SEARCH
Two modes for video input
CineSync does NOT host or proxy video (legal reasons, bandwidth). It plays from a
URL that both users have access to.
 Mode 1 — Direct URL: Host pastes any direct video URL (mp4, m3u8 HLS
 stream). Both users load the same URL in their own browser. Works with any
public stream, personal Plex/Jellyfin, or public domain films.
 Mode 2 — TMDB search UI: Search movies using the free TMDB API for
 metadata (title, poster, description, year). TMDB does NOT provide streaming —
it gives info only. Use this to pick what to watch, then user provides the streaming
URL separately.
 GET
https://api.themoviedb.org/3/search/movie
?api_key=KEY&query;=TITLE — returns title, poster, overview, release_date
 GET
https://api.themoviedb.org/3/movie/{id}
Full movie details, runtime, genres
API key: free at themoviedb.org — 1000 req/day. Store in .env as VITE_TMDB_KEY.
BACKEND API (NODE + EXPRESS)
 POST
/api/room/create
Creates a room, returns roomId (6-char code) and hostToken
 POST
/api/room/join
{"roomId"} — validates room, returns guestToken and host's current videoUrl + videoTime

GET
/api/room/:roomId
Room state: who's in it, what's playing, current timestamp
 WS
socket.io namespace: /room
All sync events live here. Server is relay-only, stateless per event.
